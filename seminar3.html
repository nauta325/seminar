01 — CONCEPT

코드를 안 봐도,
코드를 굴립니다

터미널도, 로컬 설치도 없습니다. 브라우저에서 클로드한테 말로 시키면 끝. 수강생은 코드를 직접 짜지도, 보지도 않습니다.

THE WORKFLOW

"우리 학원 이름으로 바꿔줘", "퀴즈 10문제로 늘려줘"
— 말하면, 바뀝니다.

클로드가 GitHub 레포의 코드를 직접 수정·푸시하면, Vercel이 자동으로 재배포합니다. 수강생이 할 일은 변경을 승인(머지)하는 것뿐.

클로드(웹)에 말함→ 코드 수정·푸시→ Vercel 자동 재배포→ 끝
02 — WHY · 왜 코드로 넘어오나

앱스크립트로는
안 되는 영역이 있습니다

04 — CURRICULUM · 당일 만드는 것들

내 레포, 내 배포,
내 원서 시스템

완성 템플릿에서 시작해, 클로드로 내 학원에 맞게 고치고, 진짜 작동하는 주소(URL)까지 손에 넣고 돌아갑니다.

  • 셋업
    계정 4개 연결 — GitHub · Claude · Vercel · Supabase
    템플릿 레포를 내 소유로 복제하고, 클로드(웹)에 레포를 연결, Vercel로 자동 배포를 켜고, Supabase 데이터베이스를 만듭니다. 한 번만 하면 끝나는 셋업.
    제일 막히는 곳
    환경변수 입력 — 스크린샷으로 단계별로 떠먹여 드려, 초보도 통과합니다.
    끝나면
    내 소유의 레포 + 내 주소(URL)가 생기고, 푸시하면 자동 배포되는 파이프라인 완성.
  • 하이라이트
    바이브 코딩 — 말로 시키면 코드가 바뀐다
    "학원 이름 바꿔줘", "메인 색을 우리 브랜드로", "퀴즈를 10문제로" — 클로드한테 말로 시키고, 바뀐 코드가 자동 배포되는 걸 직접 확인합니다. 코드를 안 봐도 됩니다.
    핵심 자산
    CLAUDE.md — 클로드가 프로젝트를 이해하게 해주는 안내문. 초보가 막 시켜도 안전하게 고쳐줍니다.
    끝나면
    앞으로 기능이 필요할 때마다 말로 시켜서 직접 늘릴 수 있게 됩니다.
  • 하이라이트
    원서 시스템 — 읽기 · 퀴즈 · 진도 관리
    학생이 로그인해 원서를 읽고, 퀴즈를 풀고, 진도가 자동으로 쌓이는 시스템. 여러 학생이 동시에 풀어도 빠르게 저장됩니다 — 앱스크립트로는 안 되던 그 부분.
    만드는 것
    학생 로그인 · 원서별 퀴즈 · 진도/점수 자동 누적 · 선생님 확인 화면
    끝나면
    학생들이 자기 속도로 원서를 읽고 푸는 동안, 원장은 진도를 한눈에 봅니다.
  • 운영
    내 학원에 맞게 — 커스터마이즈 & 확장
    브랜드 색·문구·퀴즈 난이도를 내 학원에 맞춰 바꾸고, 앞으로 기능을 어떻게 더 붙일지(말로 시키는 법)까지 익힙니다. 세미나 후에도 혼자 굴릴 수 있게.
    비용 안내
    Vercel·Supabase 무료 티어로 학원 1곳 30명은 여유. (Claude 유료 플랜은 필요)
    끝나면
    막히면 클로드한테 물어 고치는 습관이 들어, 더는 외주가 필요 없습니다.

06 — FAQ

자주 묻는 질문

코딩을 정말 몰라도 되나요?

네. 코드를 직접 짜지도, 보지도 않습니다. 완성 템플릿에서 시작해, 클로드한테 말로 시켜서 내 학원에 맞게 고치는 방식이에요. 배포도 자동입니다.

세미나 1·2를 안 들었어도 되나요?

1·2차보다 상위 코스라, 노션·자동화 개념을 알고 오시면 훨씬 매끄럽습니다. 동급의 경험이 있다면 바로 들으셔도 괜찮아요.

비용이 많이 드나요?

Vercel·Supabase는 무료 티어로 학원 1곳 30명 정도는 여유 있게 돌아갑니다. 다만 클로드로 코드를 고치려면 Claude 유료 플랜이 필요해요. 이 부분은 미리 안내해 드립니다.

앱스크립트(2차)랑 뭐가 다른가요?

2차는 한 명씩 순차 처리되는 자동화에 강합니다. 3차는 여러 학생이 동시에 풀고 저장하는 원서 퀴즈처럼, 동시접속·속도가 필요한 시스템을 만듭니다.

세미나 후에 혼자 고칠 수 있을까요?

그게 목표입니다. 막히면 클로드한테 말로 물어 고치는 습관을 들이도록 구성했어요. CLAUDE.md라는 안내문이 클로드를 똑똑하게 만들어, 초보가 시켜도 안전하게 고쳐줍니다.

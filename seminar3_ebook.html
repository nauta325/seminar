PART 0 · 준비
왜 만드나 — 원서 시스템의 그림
기능 설치에 들어가기 전에, 우리가 오늘 무엇을·왜 만드는지 그림을 먼저 그립니다. 이 그림이 있으면 뒤의 모든 단계가 "지금 어디쯤인지" 보입니다.
반복 노동사서 쓰기 vs 만들기네 개의 도구오늘의 여정
0.1
원서 한 권, 얼마나 손이 가나요?

원서 한 권을 수업하려면 손이 이만큼 갑니다 — 책 고르기 → 본문 준비 → 단어 정리 → 퀴즈 만들기 → 학생별 배정 → 녹음 받기 → 채점 → 엑셀에 점수 정리. 한 권에 30분에서 며칠씩 걸리고, 학생이 많아질수록 늘어납니다.

⚠️ 특히 번거로운 것 — 녹음학생마다 카톡으로 음원을 따로 받고, 누가 보냈는지 일일이 확인하고, 받았다고 표시하고… 매번 반복이죠. 이 시스템에서는 학생이 올린 녹음이 한곳에 자동으로 모이고, 제출 여부도 자동 표시되고, 클릭 하나로 바로 재생됩니다.
0.2
사서 쓰기 vs 만들어 쓰기

시중의 유명 원서 프로그램(예: 르네상스/AR)은 초기 세팅비 수백만 원에, 매달도 수백만 원씩 나갑니다. 게다가 책·레벨은 그 회사가 정한 대로만 써야 하고, 해지하면 데이터도 사라지죠. 우리가 만드는 시스템은 반대입니다.

항목사서 쓰는 프로그램우리가 만든 시스템
비용세팅 수백만 + 월 수백만 원직접 제작 · 추가 비용 ≈ 0
책정해진 라인업만우리가 쓰는 책 자유 추가
레벨·시리즈회사 기준우리 학원 기준 그대로
녹음카톡으로 받고 수기 체크자동 수집·표시·클릭 재생
데이터그 회사 서버에 갇힘우리 데이터는 우리 것
변경요청해도 안 됨말하면 그날 바뀜
📘 이 교재의 전제코드를 짜는 법을 배우는 게 아닙니다. 이미 다 돌아가는 앱을 내 것으로 만들고, 클로드에게 말해서 고치는 법을 배웁니다. 그래서 코딩을 몰라도 괜찮습니다.

PART 2 · 준비
개념은 딱 한 번 — 기술 이야기
여기서 기술 용어를 딱 한 번만 정리합니다. 이 다섯 가지 비유만 잡아 두면, 뒤에서 어떤 버튼을 누르든 "이게 왜 필요한지"가 보입니다.
저장소데이터베이스배포환경변수API 키
2.1
저장소(GitHub) = 코드 원본 창고

앱은 결국 여러 개의 파일입니다. 이 파일들을 모아 두고 버전을 관리하는 인터넷 창고가 저장소(repository)예요. 완성된 원서 앱의 저장소를 통째로 내 계정으로 복제해서 내 원본으로 삼습니다.

2.2
데이터베이스(Supabase) = 인터넷에 있는 엑셀 창고

반·학생·책·점수·녹음처럼 계속 쌓이고 바뀌는 데이터를 담는 곳입니다. 엑셀 표 여러 개가 인터넷에 있다고 생각하면 쉬워요. 앱 화면에서 학생을 추가하면 이 표에 한 줄이 쌓이고, 다시 열 때 여기서 불러옵니다.

2.3
배포(Vercel) = 내 코드를 인터넷 주소로

내 컴퓨터 안의 코드는 나만 볼 수 있죠. 배포(deploy)는 그 코드를 인터넷에 올려 누구나 접속할 주소로 만드는 일입니다. Vercel이 내 GitHub 저장소를 읽어 내주소.vercel.app 을 만들어 줍니다.

2.4
환경변수 = 코드 밖에 숨겨 두는 비밀 값

데이터베이스 주소나 API 키 같은 비밀 값을 코드 안에 적으면, 소스를 보는 누구나 볼 수 있어 위험합니다. 그래서 이런 값은 코드가 아니라 Vercel 설정(환경변수)에 따로 넣어 둡니다. 코드는 "환경변수에서 꺼내 써"라고만 되어 있어요.

⚠️ 이 앱의 절대 규칙API 키·비밀 값은 Vercel 환경변수에만 넣습니다. HTML이나 브라우저 코드에는 절대 넣지 않아요 — 학생·학부모가 소스를 볼 수 있으니까요.
2.5
API · API 키 = AI에게 일을 시키는 전화선과 사원증

API는 다른 서비스(여기선 Claude)에게 일을 시키는 통로예요. 우리 앱이 "이 지문으로 퀴즈 만들어 줘"라고 Claude에게 요청을 보내면, Claude가 결과를 돌려줍니다. API 키는 그 통로를 쓸 수 있는 사원증이에요. 키가 있어야(그리고 잔액이 있어야) 요청이 통합니다.

2.6
다섯 개가 어떻게 이어지나
GitHub
(코드 원본)
→
Vercel
(배포·실행)
↔
Supabase
(데이터)

Vercel이 GitHub의 코드를 실행하고, 그 코드가 Supabase에서 데이터를 읽고 씁니다. AI가 필요할 땐 Vercel의 코드가 환경변수에 든 API 키로 Claude를 호출합니다.

✅ PART 2 요약
  • 저장소=코드 창고, 데이터베이스=엑셀 창고, 배포=인터넷 주소 만들기.
  • 환경변수=코드 밖에 숨긴 비밀 값. 키는 절대 코드에 안 넣는다.
  • API 키=AI에게 일을 시키는 사원증. 잔액이 있어야 통한다.
  • Vercel이 GitHub 코드를 실행하고 Supabase 데이터를 다룬다.

PART 4 · 설치
Supabase — 데이터베이스 구축
데이터가 쌓일 창고를 만듭니다. 프로젝트를 만들고 → SQL 한 번으로 표·저장공간을 자동 생성하고 → 연결에 쓸 값 두 개를 복사합니다.
프로젝트 생성SQL 실행표 8종키 복사
4.1
🖐️ 새 프로젝트 만들기
  • Supabase 로그인 → New project
  • 프로젝트 이름 입력, Database Password 설정 (메모해 두기)
  • Region은 Northeast Asia (Seoul) 추천 → Create new project
  • 1~2분간 준비되는 동안 기다립니다
💡 비밀번호는 메모데이터베이스 비밀번호는 나중에 필요할 수 있으니 메모장에 적어 두세요. (배포에는 이 비번이 아니라 4.5의 키를 씁니다.)
4.2
🖐️ SQL 한 번으로 표·저장공간 생성
  • 왼쪽 메뉴 SQL Editor → New query
  • 내 저장소의 sql/mini-setup.sql 내용을 전체 복사
  • 편집기에 붙여넣고 오른쪽 아래 Run 클릭
  • 표·녹음 저장공간·샘플 책이 한 번에 만들어집니다
📘 여러 번 실행해도 안전이 SQL은 "이미 있으면 무시"하도록 만들어져 있어요. 실수로 두 번 돌려도 데이터가 깨지지 않습니다. 나중에 앱을 업데이트할 때 다시 돌려도 안전해요.
4.3
📘 만들어지는 표 8종 (무엇이 어디에 쌓이나)
표 이름담는 것
mini_books책 + 3종(단어카드 words · 원서퀴즈1 q1 · 원서퀴즈2 q2) + 원문·챕터·영상
mini_classes반
mini_students학생 (이름·반·로그인 아이디·레벨)
mini_assignments배정 (학생 ↔ 책, 중복 방지)
mini_results결과·점수 (quiz_type 0=단어·1=퀴즈1·2=퀴즈2)
mini_recordings녹음 메타 (실제 파일은 recordings 버킷)
mini_worksheets저장한 워크시트 (리딩·라이팅·단어)
mini_settings앱 설정 (예: 영어 출제 시작 레벨)
4.4
📘 recordings 버킷 (녹음 파일 저장소)

SQL은 표뿐 아니라 recordings라는 비공개 저장 버킷도 만듭니다. 학생이 녹음한 오디오 파일은 여기 저장되고, 표(mini_recordings)에는 "누가·어떤 책을·어디에" 저장했는지 정보만 남아요.

⚠️ 녹음이 안 올라가면대개 recordings 버킷이 없어서예요. 4.2의 SQL을 다시 실행하면 버킷까지 생성됩니다.
4.5
🖐️ 연결에 쓸 값 2개 복사
  • 왼쪽 Settings → API 이동
  • Project URL 복사 → 배포 때 SUPABASE_URL 로 씀
  • service_role 키 복사 → 배포 때 SUPABASE_SERVICE_KEY 로 씀
⚠️ service_role은 최고 권한 비밀키이 키는 데이터베이스를 마음대로 다룰 수 있는 강력한 키예요. 다음 파트에서 Vercel 환경변수에만 넣고, 카톡·화면공유·HTML에는 절대 붙여넣지 마세요.
✅ PART 4 요약
  • 새 프로젝트 → SQL 한 번 실행 → 표 8종·recordings 버킷·샘플 책 자동 생성.
  • SQL은 여러 번 돌려도 안전하다.
  • Settings → API에서 Project URL과 service_role 키를 복사해 둔다.
  • service_role 키는 Vercel 환경변수에만 넣는다.
PART 5 · 설치
Vercel — 인터넷에 올리기(배포)
내 저장소를 Vercel에 연결하고, 앞서 복사한 값들을 환경변수로 넣은 뒤 배포합니다. 끝나면 진짜 내 주소가 생깁니다.
Import환경변수 3개Deploy자동 재배포
5.1
🖐️ 내 저장소 불러오기(Import)
  • vercel.com → Add New → Project
  • PART 3에서 만든 내 저장소를 Import
  • 빌드 설정은 기본값 그대로 (건드릴 것 없음)
5.2
🖐️ 환경변수 3개 입력 (가장 중요)

배포 화면의 Environment Variables에 아래 3개를 추가합니다. 이름은 정확히 똑같이, 값은 앞서 복사한 것으로.

이름 (Key)값 (어디서 온 값)필수
SUPABASE_URLSupabase Settings→API의 Project URL (4.5)✅
SUPABASE_SERVICE_KEYSupabase의 service_role 키 (4.5)✅
ANTHROPIC_API_KEYClaude 콘솔의 API 키 (1.5)선택
💡 ANTHROPIC_API_KEY 없이도 배포는 돼요없으면 "AI로 생성" 대신 수동 "책 추가"만 쓸 수 있어요. 세미나에서는 넣고 진행합니다. 나중에 Vercel 설정에서 추가하고 재배포해도 됩니다.
5.3
🖐️ Deploy → 내 주소 확인
  • Deploy 클릭 → 1~2분 기다리기
  • 완료되면 내프로젝트.vercel.app 주소 생성
  • 주소 뒤에 /mini/ 를 붙여 열기 → 내프로젝트.vercel.app/mini/
💡 이 화면이 뜨면 배포 성공홈 화면(학생·학생관리·원서관리 버튼)이 뜨면 끝났어요. 이제 이 주소가 우리 학원 원서 앱입니다.
5.4
📘 자동 재배포 — 고치면 저절로 반영

Vercel은 내 GitHub 저장소와 연결돼 있어요. 그래서 나중에 코드가 바뀌면(커밋·푸시) Vercel이 자동으로 다시 배포합니다. PART 8에서 클로드로 앱을 고치고 저장하면, 몇 분 뒤 내 주소에 저절로 반영돼요.

✅ PART 5 요약
  • Vercel에 내 저장소를 Import 한다.
  • 환경변수 3개(URL·service_role·API키)를 이름 그대로 넣는다.
  • Deploy 후 /mini/ 로 접속해 홈 화면을 확인한다.
  • 이후 코드가 바뀌면 자동으로 재배포된다.

8.1
📘 저장소를 클로드로 열기

내 저장소를 Claude Code로 열면, 클로드가 그 파일들을 읽고 내 지시대로 고칠 수 있어요. 여러분은 코드를 보지 않고 말로만 시킵니다. 고친 뒤 저장(커밋·푸시)하면 Vercel이 자동 재배포합니다.

8.2
📘 커스터마이징 지도 — 말하기 → 바뀌는 곳
이렇게 말하세요바뀌는 것위치(참고)
"대표 색을 분홍색으로 바꿔줘"앱 전체 색:root --brand
"통과 점수를 80점으로 바꿔줘"퀴즈 통과 기준reader.html --pass-score
"리포트 학원 이름을 우리 학원으로"리포트·워크시트 학원명var ACADEMY
"레벨 목록을 우리 학원 레벨로"레벨(단계) 목록books.html var LEVELS
"시리즈 목록을 우리 책들로"원서 시리즈 목록books.html var SERIES
"퀴즈 문항 수를 늘려줘 / 영어로 내줘"AI 생성 프롬프트·문항 수api generate
📘 위치는 몰라도 됩니다표의 "위치"는 참고일 뿐이에요. 실제로는 "통과 점수를 80점으로 바꿔줘"라고만 말하면 클로드가 알아서 그 파일을 찾아 고칩니다.
8.3
🖐️ 예시 — 이렇게 말하면 돼요
# 학원 정체성 한 번에 "우리 학원 이름을 홍보은영어로 바꾸고, 대표 색은 따뜻한 주황색으로, 통과 점수는 75점으로 바꿔줘." # 레벨·시리즈를 우리 것으로 "레벨 목록을 1단계·2단계·3단계로 바꾸고, 시리즈는 Henry and Mudge, Fancy Nancy, Nate the Great로 바꿔줘." # AI 출제 방식 조정 "원서퀴즈2를 5문항으로 늘리고, 3레벨 이상은 보기까지 전부 영어로 내줘."
8.4
📘 바뀐 뒤 — 저장하면 자동 반영

클로드가 고친 내용을 저장(커밋·푸시)하면 GitHub에 올라가고, Vercel이 자동으로 다시 배포합니다. 몇 분 뒤 내 주소를 새로고침하면 바뀐 화면이 보여요.

💡 "이거 불편한데?" 하면 그 자리에서정렬을 바꾸고 싶거나, 칸을 하나 더 넣고 싶으면 그냥 클로드에게 말하세요. 사서 쓰는 프로그램은 요청해도 안 되지만, 이건 말하면 그날 바뀝니다.
✅ PART 8 요약
  • 저장소를 Claude Code로 열고 "이렇게 바꿔줘"라고 말로 고친다.
  • 학원명·색·통과점수·레벨·시리즈·AI 출제까지 전부 바꿀 수 있다.
  • 위치를 몰라도 클로드가 알아서 파일을 찾는다.
  • 저장하면 Vercel이 자동 재배포해 몇 분 뒤 반영된다.

트러블슈팅비용백업안전자료실
9.1
🛠 트러블슈팅
이런 증상이렇게 해결
/mini/ 열었더니 데이터가 안 뜸Vercel 환경변수(URL·service_role) 오타 확인 → 고치고 재배포(Redeploy)
저장·등록이 안 됨Supabase에서 SQL을 안 돌렸을 가능성 → PART 4의 SQL 다시 실행
"AI로 생성"이 작동 안 함ANTHROPIC_API_KEY 없음/오타/잔액 부족 → Vercel 환경변수·Claude 결제 확인
녹음이 안 올라감recordings 버킷이 없을 수 있음 → PART 4의 SQL 재실행으로 버킷 생성
Claude 결제가 자꾸 실패크롬 언어를 English로 바꾸고 재시도 (설정→언어→English 맨 위→재시작)
바꿨는데 화면에 반영 안 됨Vercel 재배포 완료 확인 → 브라우저 강력 새로고침(Ctrl+Shift+R)
학생이 로그인이 안 됨등록한 이름 철자 확인 / 아이디를 넣었다면 아이디도 일치해야 함
🆘 그래도 막히면에러 메시지를 그대로 복사해서 클로드에게 붙여넣고 "이거 왜 이래?"라고 물어보세요. 대부분 바로 원인을 짚어 줍니다.
9.2
📘 비용 관리
  • Vercel · Supabase — 학원 규모에선 대개 무료 범위로 충분
  • Claude API — 사용한 만큼 과금. 원서 몇 개 생성은 몇 백 원 수준이라 부담이 적어요
  • 콘솔에서 사용량·잔액을 가끔 확인하고, 필요할 때만 소액 충전
📘 르네상스 한 달 치면 몇 년사서 쓰는 프로그램의 한 달 구독료면, 이 시스템은 몇 년을 굴리고도 남습니다.
9.3
📘 데이터는 내 것 · 안전 수칙
  • 학생·점수·녹음은 내 Supabase에 있음 — 그 회사 서버에 갇히지 않아요
  • service_role 키·API 키는 Vercel 환경변수에만. 카톡·화면공유·HTML 금지
  • 키가 노출됐다면 즉시 삭제하고 새로 발급 (Supabase·Claude 콘솔에서)
  • 테이블 이름의 mini_ 접두사는 그대로 두기 (다른 시스템과 한 Supabase를 같이 쓸 수 있어요)
9.4
📘 계속 키워 가기

이 시스템은 완성이 아니라 시작이에요. 쓰다가 불편한 점이 보이면 클로드에게 말해서 그때그때 고치고, 새 기능도 붙일 수 있어요. "매일 개선되는 살아 있는 시스템"이 사서 쓰는 프로그램과 가장 다른 점입니다.

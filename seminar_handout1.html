노션 기초 — 이것만 알면 돼요
1
노션이 뭐예요?

메모, 표, 일정, 체크리스트를 한 곳에 모아두는 클라우드 메모장이에요.

엑셀 + 메모장 + 구글 드라이브를 합쳐놨다고 생각하면 돼요.

무료로 쓸 수 있고, PC·폰 어디서나 접속 가능해요

모든 내용은 블록으로 이루어져요.
텍스트, 표, 이미지, 체크박스 모두 블록이에요.

3
꼭 알아야 할 단축키
  • / 입력 → 블록 추가 메뉴 열기
  • /database 또는 /데이터베이스 → 데이터베이스(표) 만들기
  • /h1 → 큰 제목 만들기
  • /todo → 체크박스 만들기
  • Ctrl+D → 블록 복제
  • Ctrl+Z → 실행 취소
  • Ctrl+/ → 블록 타입 변경
  • 드래그 → 블록 순서 이동
4
데이터베이스를 보는 5가지 방법 — 뷰(View)

데이터는 하나인데, 보는 방법은 여러 개예요. 같은 표를 용도에 맞게 다르게 펼쳐 보는 게 노션의 핵심이에요.

📋 표(Table)
🗂 보드(Board)
📅 캘린더
🖼 갤러리
📃 목록(List)
  • DB 좌측 상단 뷰 이름 옆 + (보기 추가) 클릭 → 원하는 형태 선택
  • 표 — 전체 데이터를 한눈에 (기본)
  • 보드 — 칸반처럼 상태별로 카드 정리 (예: 재원/휴원)
  • 캘린더 — 날짜 속성 기준으로 달력에 표시 (예: 보강 일정)
  • 갤러리 — 카드 형태, 표지 이미지와 함께 (예: 교재 표지)
💡 뷰는 무한정 추가 가능
같은 DB에 "표 뷰 + 캘린더 뷰 + 보드 뷰"를 동시에 둘 수 있어요. 데이터는 그대로, 보는 창만 늘어나는 거예요.
5
필터 · 정렬 · 그룹화 — 원하는 것만 꺼내기
  • 필터(Filter) — 조건에 맞는 행만 보기
    예: 재원상태 = 재원 / 결제상태 = 미납
  • 정렬(Sort) — 순서대로 줄 세우기
    예: 등원시간 빠른 순 / 이름 가나다순
  • 그룹화(Group) — 같은 값끼리 묶어서 블록으로
    예: 담당선생님별 / 요일별 / 학년별
  • DB 우측 상단 ··· (점 세 개) → 필터 / 정렬 / 그룹화 메뉴
✅ 실전 예시 — 전체학생DB 하나로 "재원생만(필터) → 등원시간순(정렬) → 요일별로 묶기(그룹화)" 하면 그게 바로 시간표가 돼요. 새 DB를 안 만들어도 돼요.
⚠️ 필터·정렬·그룹화는 "뷰별로" 저장돼요
한 뷰에 필터를 걸어도 다른 뷰에는 영향 없어요. 그래서 "재원생 뷰", "미납 뷰"를 따로 만들어 둘 수 있어요.
6
버튼(Button) 만들기 — 반복 작업 한 번에

매번 똑같이 하는 작업(새 행 추가, 날짜 채우기)을 버튼 하나로 끝내요.

  • 페이지에서 /버튼 또는 /button 입력 → 버튼 블록 생성
  • 버튼 이름 입력 (예: 오늘 리포트 추가)
  • + 단계 추가 → "페이지 추가 위치" 선택 → 대상 DB 지정 (예: 데일리리포트DB)
  • 속성 자동 편집 — 날짜 = 오늘, 카톡전송 = 미전송 등 미리 채워두기
  • 완성! 이제 버튼만 누르면 값이 채워진 새 행이 자동 생성돼요
💡 버튼이 빛나는 순간
"오늘 날짜 + 미전송 상태"가 자동으로 들어간 리포트 행이 클릭 한 번에 만들어져요. 매일 반복하는 일일수록 버튼의 효과가 커요.

학원 DB 만들기 — 따라하기
1
전체학생 DB
  • 노션 새 페이지 → /database (또는 /데이터베이스) → Table — Full page 선택
  • 제목: 전체학생DB
  • 속성 추가: 학생ID(텍스트), 재원상태(선택), 학교(선택), 학년(선택), 담당선생님(선택), 전화번호(전화번호)
  • 요일별 등원시간: 월-등원시간 ~ 금-등원시간 (선택 타입 × 5개)
  • 교재 연결(관계형): 리딩교재 / 단어교재 / 문법교재 → 교재DB 연결
💡 시간표 = 전체학생DB의 뷰
별도 DB가 아니에요. 전체학생DB에서 + 보기 추가 → 그룹화 → 월-등원시간 선택하면 시간대별로 자동 분류돼요.
2
교재 DB
  • 새 페이지 → 교재DB
  • 속성: 교재명(제목), 종류(선택: 리딩/문법/단어), 가격(숫자), 누적교재수량(숫자)
  • 교재비관리DB 연결 후 → 배포된수량(롤업), 재고(수식: 누적교재수량 - 배포된수량) 추가
3
교재비관리 DB
  • 새 페이지 → 교재비관리DB
  • 속성: 학생이름(관계형→전체학생DB), 배부교재(관계형→교재DB), 배부날짜(날짜), 배부월(선택)
  • 결제상태(선택: 미납/완납), 카톡전송(선택: 미전송/전송완료)
  • 자동 계산: 교재비합계(롤업→배부교재→가격→합계), 전화번호(롤업→학생이름→전화번호)
  • Make 문자용 수식: 이름값(format(학생이름)), 배부교재값(format(배부교재))
4
데일리리포트 DB
  • 새 페이지 → 데일리리포트DB
  • 속성: 학생이름(관계형→전체학생DB), 날짜(날짜)
  • 수업 상태: 지각 / 태도 / 과제 / 단어테스트_결과 (선택)
  • 교재 연결: 리딩교재 / 단어교재 / 문법교재 (관계형→교재DB)
  • 진도: 리딩진도 / 단어진도 / 문법진도 / 추가사항 (텍스트)
  • 카톡전송(선택: 미전송/전송완료) ← Make 트리거!
  • 롤업: 진화번호(학생이름→전화번호)
  • 수식(Make 문자용): 학생이름값, 리딩교재값, 문법교재값, 단어교재값 (format(각 관계형))
⚡ 수식 속성이 왜 필요한가요?
관계형 속성은 Make에서 바로 텍스트로 쓸 수 없어요. format(리딩교재) 수식을 만들어두면 Make에서 문자 내용에 바로 넣을 수 있어요.

5
출결·보강 DB — 하나의 DB, 세 가지 뷰

결석과 보강을 한 DB로 관리하고, 뷰만 바꿔서 "오늘 보강 / 이번주 보강 / 이번달 결석"을 꺼내봐요.

  • 새 페이지 → 출결보강DB
  • 속성: 학생(관계형→전체학생DB), 구분(선택: 결석/보강), 날짜(날짜)
  • 사유(선택: 병결/개인사정/학교행사…), 상태(선택: 예정/완료), 메모(텍스트)
  • 담당선생님: 롤업(학생 → 담당선생님)으로 자동으로 가져오기

▶ 같은 DB에 뷰 3개 만들기

📅 오늘 보강
구분=보강 · 날짜=오늘
📅 이번주 보강
캘린더 뷰 · 다음 7일
📋 이번달 결석
구분=결석 · 이번 달
  • 오늘 보강 뷰 — 표 뷰 + 필터: 구분 = 보강 AND 날짜 = 오늘
  • 이번주 보강 뷰 — 캘린더 뷰 + 필터: 구분 = 보강 · 날짜 = 다음 7일
    "일주일 뒤 누가 보강인지" 달력으로 한눈에 보여요
  • 이번달 결석 뷰 — 표 뷰 + 필터: 구분 = 결석 · 날짜 = 이번 달
💡 왜 캘린더 뷰가 좋을까?
보강은 "미래 일정"이라 표보다 달력이 직관적이에요. 며칠 뒤 보강이 몰려 있는지 한눈에 보여서 수업 준비가 쉬워져요.
6
원장 스케줄 캘린더 — 내 하루를 한 곳에

상담·행정·수업·개인 일정을 한 DB에 모으고 캘린더 뷰로 봐요. 분류별로 색을 다르게 하면 하루가 한눈에 정리돼요.

  • 새 페이지 → 원장일정DB
  • 속성: 제목(제목), 날짜(날짜), 분류(선택: 상담/행정/수업/개인), 메모(텍스트)
  • + 보기 추가 → 캘린더 선택 → 날짜 속성 기준으로 표시
  • 캘린더 우측 ··· → 색 지정 → 분류 기준 → 분류별로 다른 색

CHAPTER 2 · 심화
관계형 · 롤업 · 수식 — 노션의 핵심 3가지
1
관계형(Relation) — DB끼리 연결하기

두 데이터베이스를 서로 연결하는 기능이에요. "이 리포트는 어느 학생 것인가"를 연결해두면, 학생 정보를 그 리포트에서 끌어다 쓸 수 있어요.

데일리리포트DB
↔
전체학생DB
  • 속성 추가 → 타입 관계형(Relation) 선택 → 연결할 DB 고르기
  • 예: 데일리리포트DB의 학생이름 속성을 전체학생DB와 관계형 연결
  • 이제 리포트마다 "어느 학생인지" 지정 가능, 학생 정보도 따라옴
💡 관계형이 모든 것의 시작
롤업도, 자동화도 관계형 연결이 있어야 작동해요. "학생이름"을 관계형으로 연결하는 게 첫 단추예요.
2
롤업(Rollup) — 연결된 DB에서 값 가져오기

관계형으로 연결한 DB에서 특정 값을 끌어오는 기능이에요. 직접 입력 안 해도 연결된 곳의 정보가 자동으로 들어와요.

  • 속성 추가 → 타입 롤업(Rollup) 선택
  • 관계형: 어떤 연결을 통해 가져올지 (예: 학생이름)
  • 속성: 그 DB의 어떤 값을 가져올지 (예: 전화번호)
  • 계산: 원본 그대로 / 합계 / 개수 등
예시 1 — 전화번호 가져오기
데일리리포트에서 학생 전화번호를 직접 안 적어도, 롤업으로 전체학생DB의 전화번호를 자동으로 끌어와요. (Make 문자 발송에 사용)
예시 2 — 교재비 합계
교재비관리DB에서 배부한 교재들의 가격을 합계로 롤업하면 총 교재비가 자동 계산돼요.
3
수식(Formula) — 값을 계산·변환하기

엑셀 함수처럼 계산하거나 형식을 바꾸는 기능이에요. 학원 자동화에서 특히 중요한 게 format() 함수예요.

⚡ 왜 format()이 필요할까?
관계형 속성은 Make에서 바로 텍스트로 못 써요 (내부적으로 ID로 저장되기 때문). format(관계형속성)으로 감싸면 사람이 읽는 텍스트로 바뀌어서 Make 문자에 넣을 수 있어요.
📋 데일리리포트DB — 복사용 수식 모음
format(학생이름)
format(리딩교재)
format(문법교재)
format(단어교재)

각 수식 속성 이름은 학생이름값, 리딩교재값처럼 "값"을 붙여 만들면 구분하기 좋아요.

📋 교재비관리DB — 복사용 수식
format(학생이름)
format(배부교재)
📋 교재DB — 재고 계산 수식
누적교재수량 - 배포된수량

SMS와 알림톡, 뭐가 다를까?

학원 안내는 SMS로도, 카카오톡 알림톡으로도 보낼 수 있어요. 둘의 차이를 알면 학원에 맞는 걸 고를 수 있어요.

항목 📱 SMS 💛 알림톡
학부모 입장에서 "또 광고인가" 무시 "학원 공지왔네" 즉시 확인
도달률 약 95% 99%+ (실패 시 SMS 폴백)
길이 SMS 90자 / LMS 2000자 최대 1000자
건당 비용 7~30원 8~9원 (LMS보다 저렴!)
사전 준비 발신번호 등록 채널 + 템플릿 심사
💡 학원에 알림톡이 잘 맞는 이유
매일 가는 안내(진도·결석·교재비)는 "정보 전달"이라 카카오가 좋아하는 카테고리예요. 길어도 30원이던 LMS가 알림톡에선 8~9원이라 장기적으로 통신비도 절약됩니다.
1
사전 준비물 체크 (사전 준비 페이지에서 했어요)
  • 카카오톡 비즈니스 채널 개설 — center-pf.kakao.com에서 학원 채널 만들기 (검색용 ID 필수)
  • 솔라피 → 카카오 알림톡 메뉴에서 발신 프로필 등록 (방금 만든 채널의 검색용 ID로 인증)
  • 솔라피 발신번호 승인 — SMS 폴백용으로 그대로 사용
✅ 확인하는 법 — 솔라피 로그인 → "카카오 알림톡" → "발신 프로필"에 내 학원 채널이 활성 상태로 보이면 OK!
2
알림톡 템플릿 등록 — 가장 중요한 단계

알림톡은 "이런 내용으로 보낼 거예요"를 카카오에 미리 신고하고 심사 받아야 발송할 수 있어요. SMS와의 가장 큰 차이.

  • 솔라피 로그인 → 카카오 알림톡 → 템플릿 관리 클릭
  • 우상단 "새 템플릿 등록" 버튼
  • 발신 프로필 선택 — 사전 준비에서 등록한 학원 채널
  • 템플릿 이름 입력 (관리용. 예: "데일리리포트")
  • 카테고리: 교육 → 수업/강좌 선택
  • 메시지 유형: 기본형 (가장 단순한 형태)
  • 본문 작성 — 변수는 #{변수명} 형식! (SMS는 {{}}, 알림톡은 #{})
  • (선택) 강조표기 — 첫 줄을 굵게 강조 가능
  • 심사 신청 → 카카오에서 1~3일 검토
⏰ 심사 기간 1~3일 — 미리미리
템플릿 등록부터 승인까지 영업일 기준 1~3일이 걸려요. 세미나 당일 등록하면 며칠 기다려야 자동화 켤 수 있어요.
❗ 광고성 문구는 거부됨
"할인 / 무료 / 이벤트 / 신규생 모집" 같은 광고 단어 들어가면 카카오 심사 거부됩니다. 정보 전달성 문구로만 작성하세요. 거부되면 수정해서 다시 신청 가능.
3

SMS 자동 폴백 — 강력 추천 ★

알림톡은 받는 사람이 카카오톡 안 쓰거나 / 채널 차단했거나 / 알림 꺼두면 발송 실패해요. 이럴 때 자동으로 SMS로 전환되게 설정하면 — 학부모는 무조건 받습니다.

  • Make의 Alimtalk 모듈 설정에서 "Fallback Message" 옵션 ON
  • Fallback Type: LMS (긴 문자) 선택
  • Fallback 본문 — 알림톡 본문과 비슷하게 작성하되, #{변수} 대신 노션 데이터 직접 매핑
    예: "#{학생이름}" → "1. 학생이름값.string" 끌어다 놓기
  • Fallback 발신번호: 솔라피 등록 번호 (기존과 동일)
  • 저장
✅ 폴백 작동 시나리오
① 알림톡 발송 시도 → 성공 = 알림톡으로 도착
② 알림톡 발송 시도 → 실패(카톡 미사용 등) → 자동으로 SMS 발송 → SMS로 도착
건당 비용은 둘 중 하나만 청구됩니다.

알림톡 특유의 에러 & 해결
증상 해결 방법
"템플릿 매칭 실패" 에러 Make에서 입력한 메시지와 솔라피에 등록된 템플릿이 한 글자라도 다르면 거부돼요. 띄어쓰기·줄바꿈까지 똑같이. 가장 안전한 건 솔라피 템플릿 화면에서 본문을 복사해 Make에 그대로 붙여넣기.
카카오 심사 거부 사유 확인 → 대부분 "광고성"으로 분류된 거예요. 할인·무료·이벤트·신규생 모집 같은 단어 제거하고 정보 전달성으로 수정 → 재신청.
발송됐는데 학부모가 못 받음 학부모가 학원 카카오 채널을 차단했을 가능성. SMS 폴백이 켜져있으면 자동으로 SMS로 들어가요. 폴백 안 켰다면 → Card 6 보고 켜기.

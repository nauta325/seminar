노션 기초 — 이것만 알면 돼요
1
노션이 뭐예요?

메모, 표, 일정, 체크리스트를 한 곳에 모아두는 클라우드 메모장이에요.

엑셀 + 메모장 + 구글 드라이브를 합쳐놨다고 생각하면 돼요.

무료로 쓸 수 있고, PC·폰 어디서나 접속 가능해요
2

페이지

모든 내용은 블록으로 이루어져요.
텍스트, 표, 이미지, 체크박스 모두 블록이에요.

3
꼭 알아야 할 단축키

  • /h1 → 큰 제목 만들기
  • /todo → 체크박스 만들기
  • Ctrl+D → 블록 복제
  • Ctrl+Z → 실행 취소
  • Ctrl+/ → 블록 타입 변경
  • 드래그 → 블록 순서 이동

CHAPTER 2
학원 DB 만들기 — 따라하기
1
전체학생 DB
  • 노션 새 페이지 → /database (또는 /데이터베이스) → Table — Full page 선택
  • 제목: 전체학생DB
  • 속성 추가: 학생ID(텍스트), 재원상태(선택), 학교(선택), 학년(선택), 담당선생님(선택), 전화번호(전화번호)
  • 요일별 등원시간: 월-등원시간 ~ 금-등원시간 (선택 타입 × 5개)
  • 교재 연결(관계형): 리딩교재 / 단어교재 / 문법교재 → 교재DB 연결
💡 시간표 = 전체학생DB의 뷰
별도 DB가 아니에요. 전체학생DB에서 + 보기 추가 → 그룹화 → 월-등원시간 선택하면 시간대별로 자동 분류돼요.
2
교재 DB
  • 새 페이지 → 교재DB
  • 속성: 교재명(제목), 종류(선택: 리딩/문법/단어), 가격(숫자), 누적교재수량(숫자)
  • 교재비관리DB 연결 후 → 배포된수량(롤업), 재고(수식: 누적교재수량 - 배포된수량) 추가
3
교재비관리 DB
  • 새 페이지 → 교재비관리DB
  • 속성: 학생이름(관계형→전체학생DB), 배부교재(관계형→교재DB), 배부날짜(날짜), 배부월(선택)
  • 결제상태(선택: 미납/완납), 카톡전송(선택: 미전송/전송완료)
  • 자동 계산: 교재비합계(롤업→배부교재→가격→합계), 전화번호(롤업→학생이름→전화번호)
  • Make 문자용 수식: 이름값(format(학생이름)), 배부교재값(format(배부교재))
4
데일리리포트 DB
  • 새 페이지 → 데일리리포트DB
  • 속성: 학생이름(관계형→전체학생DB), 날짜(날짜)
  • 수업 상태: 지각 / 태도 / 과제 / 단어테스트_결과 (선택)
  • 교재 연결: 리딩교재 / 단어교재 / 문법교재 (관계형→교재DB)
  • 진도: 리딩진도 / 단어진도 / 문법진도 / 추가사항 (텍스트)
  • 카톡전송(선택: 미전송/전송완료) ← Make 트리거!
  • 롤업: 진화번호(학생이름→전화번호)
  • 수식(Make 문자용): 학생이름값, 리딩교재값, 문법교재값, 단어교재값 (format(각 관계형))
⚡ 수식 속성이 왜 필요한가요?
관계형 속성은 Make에서 바로 텍스트로 쓸 수 없어요. format(리딩교재) 수식을 만들어두면 Make에서 문자 내용에 바로 넣을 수 있어요.

CHAPTER 4
Make 자동화 설정 — 따라하기
0
사전 준비 — 솔라피 설정
  • solapi.com 회원가입 → 로그인
  • 좌측 메뉴 발신번호 관리 → 학원 전화번호 등록 → 승인됨 상태 확인 (1~2일 소요, 미리 해두세요!)
  • 최소 50,000원 충전 (SMS 1건 약 7~8원)
  • 우상단 계정 → API Key 관리 → API Key + API Secret 복사해두기
✅ 준비물 체크 — 발신번호 승인됨 · 포인트 50,000원 이상 · API Key 복사 완료
1
노션 ↔ Make 연결 — Internal API 3단계
PART A — 노션에서 API 토큰 발급
  • 1 노션 설정 → 왼쪽 메뉴 연결 → 하단 "연결 개발 또는 관리" 클릭
  • 2 "새 API 통합 만들기" → 이름 입력 → 생성하기
  • 3 "API 통합 설정 구성" → 액세스 토큰 복사 → 메모장에 저장
PART B — Make에서 연결 만들기
  • 1 Make → Notion 모듈 → Connection Add → Connection Type: "Notion Internal"
  • 2 Internal Integration Token: PART A 토큰 붙여넣기 → Save
PART C — DB에 연결 권한 부여
  • 1 노션 데일리리포트DB 페이지 → 우상단 ··· → 연결
  • 2 목록에서 만든 API 통합 이름 선택 → 전체학생DB, 교재비관리DB도 동일하게 반복
✅ 완료 확인 — Make 시나리오에서 Data Source ID 드롭다운에 DB 이름이 보이면 성공!

Solapi 모듈 — Make에 연결
  • 시나리오에서 Solapi 검색 → Send a Text Message 선택 → Connection Add
  • API Key: 솔라피 대시보드 → 계정 → API Key 관리에서 복사한 키
  • API Secret: 같은 화면의 API Secret 붙여넣기 → Save
  • 수신번호: 1. Properties Value: 진화번호.Array[]: phone_number
  • 발신번호: 솔라피에 등록·승인된 내 전화번호 직접 입력
⚠️ 수신번호 매핑 주의 — 텍스트 직접 입력이 아니라, 왼쪽 Notion 모듈에서 드래그로 끌어다 연결해야 해요.
4
테스트 & 활성화
  • Make 하단 Run once → 데일리리포트DB에 테스트 데이터 입력 → 카톡전송 미전송 상태로 저장
  • 내 폰으로 문자 오는지 확인! 왔으면 카톡전송이 전송완료로 바뀌었는지도 확인
  • 잘 되면 시나리오 토글 켜서 활성화 (15분 간격 추천)
✅ 중복 방지 원리 — 문자가 발송되면 카톡전송이 "전송완료"로 바뀌어요. Make는 "미전송"만 가져오니까 같은 줄이 두 번 발송되는 일이 없어요.

TROUBLESHOOTING
자주 막히는 에러 & 해결법
❗
이런 에러, 이렇게 해결하세요
증상 해결 방법
DB가 드롭다운에 안 보여요 PART C — 노션 DB 페이지에서 ··· → 연결 허용을 안 했어요. DB마다 반복해야 해요.
문자가 안 와요 (에러 없음) 날짜 필터에 Asia/Seoul이 빠졌거나, 오늘 날짜 데이터가 없는 경우예요.
발신번호 오류가 나요 솔라피 발신번호 관리에서 상태가 승인됨인지 확인. 미승인이면 1~2일 대기해야 해요.
같은 학생한테 문자가 2번 와요 Update 모듈(카톡전송→전송완료)이 제대로 연결됐는지 확인. Item ID에 Map ON 해야 해요.
문자 내용에 변수가 그대로 보여요
(예: {{날짜.start}})		 이전 모듈에서 매핑이 안 된 거예요. Solapi 모듈 열어서 변수를 드래그로 다시 연결하세요.
포인트 부족 에러 솔라피 → 충전하기. 최소 50,000원 충전해야 해요.
Make가 DB 변경을 못 감지해요 시나리오 활성화(토글 ON)를 확인하고, 스케줄을 15분으로 설정했는지 확인하세요.

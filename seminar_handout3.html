CHAPTER 1
오늘 무엇을 만드나요?
1
완성된 앱에서 시작해요

바닥부터 코딩하는 게 아니라, 이미 다 돌아가는 원서 앱을 내 저장소로 복제해서 시작해요. 여러분이 할 일은 복제 → 배포 → 내 값만 교체 세 가지뿐입니다.

코드를 짜는 수업이 아니라, 완성 앱을 내 것으로 만들고 · 클로드에게 말해서 고치는 수업이에요.
2
세 조각이 합쳐져 돌아가요

GitHub는 앱 코드 보관, Supabase는 데이터(반·학생·책·점수·녹음) 저장, Vercel은 그 코드를 인터넷 주소로 올려주는 곳이에요.

이 셋을 오늘 순서대로 연결하면 내주소.vercel.app/mini/ 가 생깁니다.

CHAPTER 3
GitHub — 내 저장소 만들기
1
템플릿을 내 계정으로 복제
  • 아래 템플릿 저장소 주소를 크롬에서 열어요 (⑩ 자료실 링크)
  • 우측 상단 초록색 "Use this template" → "Create a new repository" 클릭
  • Repository name 에 원하는 이름 입력 (예: my-reader)
  • 공개(Public) 그대로 두고 "Create repository" 클릭
  • 내 계정에 똑같은 앱이 복제되면 성공!
💡 "Use this template" 버튼이 안 보이면
대신 우측 상단 "Fork" 버튼을 눌러도 똑같이 내 계정으로 복사돼요.
✅ 여기서 만든 저장소가 앞으로 Vercel이 배포할 "내 원서 앱의 원본"이에요. 이 저장소 주소를 잘 기억해두세요.

CHAPTER 4
Supabase — 데이터 창고 만들기
1
새 프로젝트 만들기
  • supabase.com 로그인 → New project 클릭
  • 프로젝트 이름·데이터베이스 비밀번호 입력 (비번은 메모해두세요)
  • Region은 Northeast Asia (Seoul) 추천 → Create new project
  • 1~2분 준비되는 동안 기다려요
2
표·저장공간을 한 번에 생성 (SQL 실행)
  • 왼쪽 메뉴 SQL Editor → New query 클릭
  • 템플릿의 sql/mini-setup.sql 파일 내용을 전체 복사
  • SQL 편집기에 붙여넣고 오른쪽 아래 Run 클릭
  • 표·녹음 저장공간·샘플 책이 한 번에 만들어져요
# 이 SQL 한 번으로 만들어지는 것 mini_books 책 + 3종 퀴즈(단어카드·원서퀴즈1·2) mini_classes 반 mini_students 학생 mini_assignments 배정(학생 ↔ 책) mini_results 퀴즈 결과·점수 mini_recordings 녹음 메타 + recordings 저장 버킷
3
연결에 쓸 값 2개 복사
  • 왼쪽 Settings → API 로 이동
  • Project URL 복사 → 나중에 SUPABASE_URL 로 씀
  • service_role 키 복사 → 나중에 SUPABASE_SERVICE_KEY 로 씀
🔑 service_role 키는 아주 강력한 비밀키예요. 다음 단계에서 Vercel 환경변수에만 넣고, 카톡·화면공유·HTML 코드에는 절대 붙여넣지 마세요. (학생·학부모가 소스를 볼 수 있어요.)

CHAPTER 5
Vercel — 인터넷에 올리기 배포
1
내 저장소 불러오기
  • vercel.com → Add New → Project 클릭
  • CHAPTER 3에서 만든 내 저장소를 Import
  • 배포 설정은 기본값 그대로 두면 돼요 (건드릴 것 없음)
2
환경변수 3개 입력

배포 화면의 Environment Variables에 아래를 추가해요. 이름은 정확히 똑같이, 값은 앞서 복사한 것으로.

이름 (Key)값 (Value)필수
SUPABASE_URLSupabase의 Project URL✅
SUPABASE_SERVICE_KEYSupabase의 service_role 키✅
ANTHROPIC_API_KEYClaude API 키 (AI 생성용)선택
💡 ANTHROPIC_API_KEY 없이도 배포는 돼요. 대신 그땐 "AI로 생성" 대신 수동 "책 추가"만 쓸 수 있어요. 세미나에선 넣고 진행해요.
3
Deploy → 내 주소 확인
  • Deploy 버튼 클릭 → 1~2분 기다리기
  • 완료되면 내프로젝트.vercel.app 주소가 생겨요
  • 주소 뒤에 /mini/ 를 붙여서 열어요 → 내프로젝트.vercel.app/mini/
🎉 이 화면이 뜨면 배포 성공! 이제 이 주소가 우리 학원 원서 앱이에요. 다음 장부터 안을 채워봅니다.

첫 실행 — 반·학생 만들기
1
페이지별 주소 (북마크 해두세요)
누가주소하는 일
🧒 학생/mini/배정된 책 → 단어·퀴즈 → 녹음
👩‍🏫 선생님/mini/students.html반·학생·배정·학습 현황
👩‍🏫 선생님/mini/books.htmlAI 생성·책 목록·수정
👨‍👩‍👧 학부모/mini/report.html학습 리포트 (인쇄·PDF)
👩‍🏫 선생님/mini/worksheet.html워크시트 출력
2
반 만들고 학생 등록
  • /mini/students.html 열기
  • 반 만들기 — 반 이름 입력 (예: 월수금 A반)
  • 학생 등록 — 이름·로그인 번호 입력 (준비해온 학생 3~5명)
  • 학생은 나중에 /mini/ 에서 이름 + 번호로 로그인해요
💡 아직 배정할 책이 없어요. 다음 장에서 AI로 책·퀴즈를 만들고, 이 학생들에게 배정합니다.

AI로 퀴즈 만들기 핵심
1
지문만 붙여넣으면 3종이 자동 생성
📖 영어 지문
붙여넣기
→
✨ AI 생성
→
🃏 단어카드
📝 원서퀴즈 1
📝 원서퀴즈 2
  • /mini/books.html 열기 → "AI로 생성"
  • 준비해온 영어 지문(5~10문장이면 충분)을 붙여넣기
  • 레벨·시리즈 선택하고 생성 클릭
  • 단어카드·원서퀴즈 2종이 자동으로 만들어져요 → 확인 후 저장
2
학생·반에 배정
  • 만든 책을 학생 또는 반에 배정
  • 학생 화면(/mini/)에서 그 책이 보이는지 확인
  • 학생은 단어 학습 → 소리내어 읽고 녹음 → 퀴즈 순으로 잠금이 풀려요
🎙️ 녹음은 자동으로 모여요. 학생이 올린 녹음은 학생·책별로 한곳에 정리되고, 선생님은 현황에서 클릭 한 번으로 바로 들어볼 수 있어요.
⚠️ "AI로 생성"이 안 될 때 — ANTHROPIC_API_KEY를 안 넣었거나 잔액이 없는 경우예요. Vercel 환경변수를 확인하세요. 키 없이 쓰려면 수동 "책 추가"로 직접 입력하면 됩니다.

CHAPTER 8
우리 학원에 맞추기 바이브 코딩
1
클로드에게 "말해서" 바꾸기

코드를 직접 고치지 않아요. 저장소를 Claude Code로 열고, 아래처럼 말로 시키면 그 자리에서 바뀝니다.

이렇게 말하세요바뀌는 것위치
"대표 색을 분홍색으로 바꿔줘"앱 전체 색:root --brand
"통과 점수를 80점으로 바꿔줘"퀴즈 통과 기준reader.html --pass-score
"리포트 학원 이름을 우리 학원으로"리포트·워크시트 학원명var ACADEMY
"레벨 목록을 우리 학원 레벨로"레벨(단계) 목록books.html var LEVELS
"시리즈 목록을 우리 책들로"원서 시리즈 목록books.html var SERIES
2
예시 — 이렇게 말하면 돼요
# 클로드에게 이렇게 부탁하세요 "우리 학원 이름을 홍보은영어로 바꾸고, 대표 색은 따뜻한 주황색으로, 통과 점수는 75점으로 바꿔줘." # 레벨·시리즈도 한 번에 "레벨 목록을 1단계·2단계·3단계로 바꾸고, 시리즈는 Henry and Mudge, Fancy Nancy로 바꿔줘."
💡 바뀐 뒤엔?
클로드가 고친 내용을 저장(커밋·푸시)하면, Vercel이 자동으로 다시 배포해서 몇 분 뒤 내 주소에 반영돼요.

CHAPTER 9
막힐 때 — 트러블슈팅
이런 증상이렇게 해결
/mini/ 열었더니 데이터가 안 뜸Vercel 환경변수(URL·service_role) 오타 확인 → 고치고 재배포(Redeploy)
저장·등록이 안 됨Supabase에서 SQL을 안 돌렸을 가능성. CHAPTER 4의 SQL을 다시 실행
"AI로 생성"이 작동 안 함ANTHROPIC_API_KEY 없음/오타/잔액 부족 → Vercel 환경변수·Claude 결제 확인
녹음이 안 올라감recordings 버킷이 없을 수 있어요. CHAPTER 4의 SQL 재실행으로 버킷까지 생성
Claude 결제가 자꾸 실패크롬 언어를 English로 바꾸고 재시도 (설정 → 언어 → English 맨 위 → 재시작)
바꿨는데 화면에 반영 안 됨Vercel 재배포가 끝났는지 확인 → 브라우저 새로고침(Ctrl+Shift+R)
🆘 그래도 막히면 — 에러 메시지를 그대로 복사해서 클로드에게 붙여넣고 "이거 왜 이래?"라고 물어보세요. 대부분 바로 원인을 짚어줍니다. 수업 중이라면 손 들어 주세요!
